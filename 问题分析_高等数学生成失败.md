# 高等数学题目生成失败问题分析

## 问题描述
- ✅ 大学语文：题目生成正常
- ✅ 大学英语：题目生成正常  
- ❌ 高等数学：题目生成失败（两次都失败）

## 代码流程分析

### 1. 数据选择流程（SubjectSelectFlow.tsx）
- 高等数学的步骤：`['章', '节', '知识点']`
- 选择完成后，`selected` 对象格式：`{ "章": "初等数学", "节": "代数式", "知识点": "分式运算" }`
- 这个格式与大学语文、大学英语类似，只是字段名不同

### 2. API调用流程
1. 前端发送请求到 `/api/deepseek/generate`
2. 传递参数：
   ```json
   {
     "subject": "高等数学",
     "selectedPath": {
       "章": "初等数学",
       "节": "代数式", 
       "知识点": "分式运算"
     }
   }
   ```

### 3. Deepseek API调用（lib/deepseek.ts）

#### Prompt 构建
```typescript
const prompt = `你是一位专业的专升本考试出题专家。现在需要你为${subject}科目出题。

知识点信息：
${JSON.stringify(selectedPath, null, 2)}

#### API请求参数
```typescript
{

  temperature: 0.7,

```

#### 响应处理
```typescript
// 1. 提取内容
const content = data.choices[0]?.message?.content;

// 2. 提取JSON（处理markdown代码块）
if (jsonContent.includes('```')) {
  const jsonMatch = jsonContent.match(/```(?:json)?\s*([\s\S]*?)\s*```/);
  if (jsonMatch) {

  }

// 3. 解析JSON
const parsed = JSON.parse(jsonContent) as QuestionsResponse;

if (!parsed.questions || !Array.isArray(parsed.questions) || parsed.questions.length !== 10) {

## 可能的问题原因

### 🔴 问题1：max_tokens 不足
**分析**：
- `max_tokens: 4000` 可能不足以生成10道完整的数学题目
- 数学题目通常包含：
  - 题目描述（可能较长，包含公式）
  - 4个选项（每个选项可能包含公式或计算过程）
  - 详细解析（数学题解析通常较长，包含步骤说明）
- 10道数学题的总token数可能超过4000

**证据**：
- 大学语文和大学英语的题目相对简单，4000 tokens可能够用
- 高等数学题目更复杂，需要更多tokens

### 🔴 问题2：JSON解析失败
**分析**：
- Deepseek API可能返回了不完整的JSON（因为token限制）
- 或者返回的JSON格式不正确
- 当前的错误处理会抛出通用错误，没有详细的错误信息

### 🔴 问题3：题目数量验证失败
**分析**：
- 如果API返回的题目数量不是10道，会抛出错误
- 可能因为token限制，只生成了部分题目

### 🔴 问题4：数学公式或特殊字符处理问题
**分析**：
- 数学题目可能包含特殊字符、公式等
- JSON序列化/反序列化可能有问题
- 但这个问题应该不会导致完全失败

### 方案3：放宽题目数量验证（临时方案）
```typescript
